Creating Objects

Object Literal Syntax

The simplest way to create an object is using the object literal syntax with curly braces {}.

// Creating a user object const user = { firstName: "John", lastName: "Doe", age: 28, isAdmin: false, address: { street: "123 Main St", city: "New York" } }; // Try editing this code and click Run to see the output console.log(user.firstName); // Accessing property with dot notation

Real World Application

User profiles in web applications are perfect examples of objects. Each user has properties like name, email, preferences, and address information that can be neatly organized in an object.

Constructor Functions

When you need to create multiple similar objects, constructor functions are useful.

// Product constructor function function Product(name, price, inStock) { this.name = name; this.price = price; this.inStock = inStock; this.displayInfo = function() { console.log(`${this.name} - $${this.price}`); }; } // Creating product instances const laptop = new Product("MacBook Pro", 1999, true); const phone = new Product("iPhone 13", 999, true); laptop.displayInfo(); // MacBook Pro - $1999 phone.displayInfo(); // iPhone 13 - $999

Real World Application

E-commerce systems use this pattern extensively. Each product in a store can be an object instance created from a Product constructor, making inventory management efficient.

Accessing Object Properties

Dot Notation vs Bracket Notation

There are two ways to access object properties in JavaScript:

const book = { title: "JavaScript: The Good Parts", author: "Douglas Crockford", year: 2008 }; // Dot notation - when you know the exact property name console.log(book.title); // "JavaScript: The Good Parts" // Bracket notation - useful when property name is dynamic const property = "author"; console.log(book[property]); // "Douglas Crockford" // Try accessing the year property using both notations

Pro Tip

Use bracket notation when the property name contains spaces or special characters, or when it's stored in a variable.

Real World Application

When building a dynamic UI that displays different user information based on user selection, bracket notation allows you to access properties dynamically without writing complex conditional logic.

Object Methods

Defining and Using Methods

Methods are functions that are properties of an object. They allow objects to "do things".

const bankAccount = { owner: "John Smith", balance: 1000, // Method to deposit money deposit(amount) { this.balance += amount; console.log(`Deposited $${amount}. New balance: $${this.balance}`); }, // Method to withdraw money withdraw(amount) { if (amount > this.balance) { console.log("Insufficient funds!"); return; } this.balance -= amount; console.log(`Withdrew $${amount}. New balance: $${this.balance}`); } }; // Try making some deposits and withdrawals bankAccount.deposit(500); // Deposited $500. New balance: $1500 bankAccount.withdraw(200); // Withdrew $200. New balance: $1300

Real World Application

Banking applications use objects with methods like this to handle account transactions. Each account is an object with properties (balance, owner) and methods (deposit, withdraw, transfer).

The 'this' Keyword

this refers to the current object context. It's crucial for object methods.

const shoppingCart = { items: [], addItem(product, quantity) { this.items.push({ product, quantity }); console.log(`Added ${quantity} ${product}(s) to cart`); }, calculateTotal() { return this.items.reduce((total, item) => { return total + (item.quantity * 10); // $10 per item }, 0); } }; // Try adding some items and calculating the total shoppingCart.addItem("Apple", 3); shoppingCart.addItem("Banana", 2); console.log(`Total: $${shoppingCart.calculateTotal()}`); // Total: $50

Important Note

Avoid using arrow functions for methods because they don't have their own this context. Use regular function syntax instead.

Real World Applications

E-commerce Product System

A complete e-commerce product management system using objects.

class Product { constructor(id, name, price, inventory) { this.id = id; this.name = name; this.price = price; this.inventory = inventory; } applyDiscount(percent) { this.price *= (1 - percent/100); } } class ShoppingCart { constructor() { this.items = []; } addItem(product, quantity) { if (quantity > product.inventory) { console.log(`Not enough ${product.name} in stock`); return; } this.items.push({ product, quantity }); product.inventory -= quantity; } getTotal() { return this.items.reduce( (total, item) => total + (item.product.price * item.quantity), 0 ); } checkout() { console.log(`Your total is $${this.getTotal()}`); console.log("Thank you for your purchase!"); this.items = []; } } // Try creating products and using the shopping cart const laptop = new Product(1, "Laptop", 999, 10); const phone = new Product(2, "Phone", 699, 15); const cart = new ShoppingCart(); cart.addItem(laptop, 1); cart.addItem(phone, 2); cart.checkout();

Common Mistakes

Mistake Problem Solution
Forgetting commas Syntax errors when separating properties Always use commas between properties
Using arrow functions for methods this won't reference the object Use regular function syntax
Modifying objects while iterating Can cause unexpected behavior Create a copy of keys first
Deep copying objects Assignment creates a reference, not a copy Use JSON.parse(JSON.stringify(obj)) or spread operator
